2. O que é Git (conceito)

  • Git é um sistema de controle de versão distribuído criado por Linus Torvalds (2005). Permite rastrear alterações, colaborar e manter o histórico do projeto. :contentReference[oaicite:4]{index=4}
  • Diferente de outros sistemas: Git trata os dados como "fotos" (snapshots) do sistema de arquivos a cada commit — eficiente e seguro. :contentReference[oaicite:5]{index=5}
  • Operações são, em grande parte, locais (rápidas). Git usa checksums para garantir integridade. :contentReference[oaicite:6]{index=6}

3. Três estados do Git (fluxo básico)

  1. Modified — você editou arquivos no diretório de trabalho.
  2. Staged (área de preparo/index) — você adicionou mudanças com git add.
  3. Committed — as alterações estão salvas no banco local com git commit.

Fluxo típico: editar → git add → git commit → git push (remoto). :contentReference[oaicite:7]{index=7}

4. Comandos principais (explicado passo a passo)

git initInicializa repositório Git local na pasta atual.
git clone <URL>Clona repositório remoto.
git statusMostra arquivos modificados / staged / untracked.
git add <arquivo>Prepara arquivo para commit (staging).
git commit -m "mensagem"Cria commit com a mensagem informada.
git logMostra histórico de commits.
git pullPuxa alterações do remoto e tenta mesclar.
git pushEnvia commits locais para o remoto.
git branchLista branches locais.
git checkout <branch>Muda para a branch informada.
git checkout -b <branch>Cria e muda para nova branch.

5. Branches — como funciona

  • Uma branch é apenas um ponteiro para um commit (não uma cópia física do código). Por padrão existe a branch main (ou master). :contentReference[oaicite:9]{index=9}
  • Exemplo de criação e fluxo: 
    # criar branch e mudar para ela
git branch dev
git checkout dev

# ou em um único passo
git checkout -b feature-x
  • HEAD aponta para a branch atual (ou diretamente para um commit, em detached HEAD). :contentReference[oaicite:10]{index=10}
  • Visualizar graficamente: 
    git log --oneline --graph --all

6. Merge e conflitos

  • Merge: combina commits de outra branch na branch atual. Tipos: fast-forward (sem commit extra) ou merge commit (quando há divergência). :contentReference[oaicite:11]{index=11}
  • Conflitos: ocorrem quando duas alterações concorrentes em mesmas linhas não podem ser mescladas automaticamente. O Git marca o arquivo com indicadores (<<<<<<<, =======, >>>>>>>).
  • Como resolver:
    1. Abra o arquivo, decida o que manter (edite para ficar correto).
    2. Marque como resolvido: git add <arquivo>
    3. Finalize com: git commit -m "Resolvido conflito"

7. Redefinir / restaurar / limpeza de arquivos

  • git restore . — desfaz alterações nos arquivos modificados (volta ao último commit). Não afeta commits. :contentReference[oaicite:13]{index=13}
  • git reset --hard <commit> — redefine o repositório para o commit indicado; descarta commits e alterações locais (cuidado: operação destrutiva). :contentReference[oaicite:14]{index=14}
  • git clean -n — mostra quais arquivos não rastreados seriam removidos (simulação). :contentReference[oaicite:15]{index=15}
  • git clean -f — remove de fato arquivos não rastreados. Use com cautela. :contentReference[oaicite:16]{index=16}

8. Introdução rápida ao JSON

  • JSON = JavaScript Object Notation. Formato leve para trocar e armazenar dados semi-estruturados. Útil em APIs (cliente ↔ servidor). :contentReference[oaicite:17]{index=17}
  • Sintaxe básica: 
    {
  "nome": "Maria",
  "idade": 22,
  "ativo": true,
  "enderecos": [
    {"rua": "A", "num": 10},
    {"rua": "B", "num": 20}
  ]
}
  • Tipos: string (entre aspas), número, boolean, null, objeto ({}), vetor ([]). JSON é independente de linguagem (parsers existem em várias linguagens). :contentReference[oaicite:18]{index=18}

9. Cheatsheet — comandos mais usados (resumido)

# configuração inicial (nome + email)
git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

# clonar
git clone https://github.com/usuario/repositorio.git

# fluxo comum
git status
git add .
git commit -m "mensagem"
git pull
git push

# branches e merge
git checkout -b minha-branch
# trabalhar...
git add .
git commit -m "feat: algo"
git checkout main
git merge minha-branch

# desfazer mudanças locais
git restore arquivo.txt      # desfaz alterações no arquivo
git reset --hard HEAD~1      # volta 1 commit (atenção!)
git clean -n                 # mostrar arquivos não rastreados
git clean -f                 # apagar arquivos não rastreados
